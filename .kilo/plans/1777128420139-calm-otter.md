# Plano: Corrigir interpretação de "X parcelas de R$ Y"

## Problema

Quando o usuário diz "7 parcelas de 354,32", a IA interpreta:
- `amount = 354.32` (total)
- `numberOfInstallments = 7`
- Resultado: cada parcela = R$ 50,62 (**ERRADO**)

O correto seria:
- `amount = 7 × 354.32 = 2480.24` (total)
- `numberOfInstallments = 7`
- Resultado: cada parcela = R$ 354,32 (**CORRETO**)

Em português, "X parcelas de Y" significa **X parcelas no valor de Y cada uma**. O sistema atual trata `amount` como o valor total (e divide por N), mas a IA está passando o valor de cada parcela como se fosse o total.

## Causa raiz

1. O parâmetro `amount` em `create_expense_installment` não tem `description` — a IA não sabe se é o valor total ou o valor de cada parcela.
2. O prompt do sistema (`finance.txt`) não tem instrução alguma sobre como interpretar "X parcelas de R$ Y".

## Mudanças necessárias

### 1. `api/src/providers/ai/gemini/declarations/finance-tools.ts` (linha 251)

Adicionar `description` no parâmetro `amount` da tool `create_expense_installment`:

```typescript
amount: {
  type: Type.NUMBER,
  description: 'VALOR TOTAL da compra (soma de todas as parcelas). Ex: se sao 7 parcelas de R$ 354,32, o total e 7 * 354,32 = 2480.24',
},
```

### 2. `api/src/providers/ai/system/finance.txt` (após linha 3)

Adicionar regras explícitas sobre interpretação de parcelas:

```
REGRAS IMPORTANTES SOBRE PARCELAS:
- "X parcelas de R$ Y" significa X parcelas no valor de Y cada. O VALOR TOTAL = X * Y.
- "parcelado em X vezes de R$ Y" tambem significa que cada parcela vale Y. TOTAL = X * Y.
- "comprei algo de R$ X em Y parcelas" significa TOTAL = X, cada parcela = X / Y.
- Se o usuario disser "parcelas de Y" sem deixar claro se Y e o total ou o valor de cada parcela, PECA CONFIRMACAO.
- O parametro amount da funcao create_expense_installment e sempre o VALOR TOTAL (soma de todas as parcelas), nao o valor individual.
- Se houver duvida, SEMPRE pergunte "qual o valor total da compra?" antes de chamar a ferramenta.
```

### 3. Validar

Rodar `npm test` e `npm run build` (ou comando equivalente) para garantir que nada quebrou.

- NÃO adicionar novo campo no banco de dados.
